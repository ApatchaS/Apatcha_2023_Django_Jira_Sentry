About page

This is a about page for the integration of Sentry and Jira.

This integration allows you to link Sentry's bug tracking system and Jira's task tracking system to automatically send bugs that get into Sentry as tasks to a specific Jira project

To get started you need:

  1. Set up Webhook integration for a project in Sentry
  2. Configuring an alert rule to initiate the sending of issues via WebHook
  3. Create a Django admin user with ./manage.py createsuperuser
  4. *It may be necessary to wait for the Issue from Sentry to come into the system before the next step (Or create an entry in the Sentry table manually)
  5. Go to the Admin page and create entries in the following tables according to your Jira data:
    • Jira auths
    • Jira connections
    • Jiras
    • Jira sentry links

Also may be usefull:

  • To see all Issues coming from Sentry, follow this link
  • To delete irrelevant Sentry or Jira projects, use the following command ./manage.py delete_outdated_projects |model: {Sentry, Jira}| [days: 30 days by default] [-f --force: without warning message]
